# Vibe Coding 全流程实战工具包

把一个想法变成能跑的产品,中间四步:**调研 → PRD → 原型 → Kickoff**。
这份工具包给你 5 条通用 Prompt 和 1 份 CLAUDE.md 模板,套上你的项目就能用。

> 这份 README 是会后查阅指南。分享里没听清的、回家想再确认的,都翻这里。

---

## 这个工具包里有什么

| 文件 | 它是什么 | 什么时候用 |
|---|---|---|
| `prompts.md` | 5 条全流程提示词 | 从调研到开工,每一步都有对应 prompt |
| `CLAUDE.md` | 给 Claude Code 的入职手册 | 项目里放一份,Claude Code 每次工作前都会读 |
| `README.md` | 这份指南 | 不确定怎么用前两份时,翻这里 |

## 整体工作流

1. **想法** —— 你的起点
2. **Step 1:调研** —— Prompt 1 → Kimi / ChatGPT / Perplexity
3. **Step 2:PRD 追问** —— Prompt 2 → ChatGPT / Claude
4. **Step 3:原型** —— Prompt 3 → v0 / Lovable / Bolt / Figma Make
5. **回写 PRD** —— 原型改完后,Prompt 4 同步回 PRD
6. **Step 4:Kickoff** —— Prompt 5 → Claude Code 开工 + 自动填 CLAUDE.md
7. **功能开发循环** —— 边写边用 `#` 键沉淀新规则到 CLAUDE.md

## 准备工作

**你需要**:
- 一个产品想法(模糊也行,Prompt 2 会帮你理清)
- 一台能装软件的电脑
- 以下账号(根据预算/偏好挑):
  - 调研:Kimi(中文调研强)/ ChatGPT Plus(含 Deep Research)/ Perplexity
  - 追问 PRD:ChatGPT / Claude.ai(任选)
  - 原型:v0 / Lovable / Bolt / Figma Make / Claude Artifacts

- 视觉参考:Pinterest 或 Dribbble(免费)

**第一次跑这套流程的预期**:
- 时间:1-3 天(慢是正常的)
- 工具订阅:0-200 元/月不等

## Step 1:调研

**目标**:搞清"这事有人做吗?他们没做好哪块?我切哪个口子?"

**怎么操作**:
1. 打开 `prompts.md`,复制 Prompt 1
2. 替换 `[填写: ...]` 处的内容
3. 粘贴到 Kimi(可开 agent 集群并行)/ ChatGPT / Perplexity
4. 等结果

**耗时**:30 分钟 - 2 小时

**完成的标志**:你能用一段话说清:这个领域哪几家在做、用户最常抱怨什么、你想切哪个空白。

**容易踩的坑**:
- 拿到一堆链接就当结论了 → AI 给的"结构化结论"才是结论,链接只是依据
- 只看了一家就下定论 → 至少 3-5 家
- 完美主义停不下来 → 信息够了就走,后面发现不对再回来

---

## Step 2:PRD 苏格拉底追问

**目标**:把模糊想法变成可执行的产品需求文档(PRD)

**怎么操作**:
1. 复制 Prompt 2,替换里面的想法描述
2. 粘贴到 ChatGPT 或 Claude
3. 它会问你**一个**问题,你回答,它继续问下一个
4. 大概 5-15 轮之后,它会主动停下,输出 PRD

**关键心法**:
- 一次只问一个,认真答,不要敷衍
- 答不出来很正常,说"我不确定"也行,它会换角度再问
- 如果它问得发散,可以说"先回到核心痛点这一块"

**耗时**:30-60 分钟

**完成的标志**:拿到一份结构化 PRD,自己读一遍能讲清楚"做什么、给谁用、不做什么"。

**容易踩的坑**:
- 一上来就让它"写 PRD"而不是先追问 → 拿到的 PRD 全是 AI 的猜测
- 跳过追问、自己塞答案给它 → 你心里的盲区也跟着塞进去了

---

## Step 3:高保真原型

**目标**:把 PRD 变成可点击的界面,看到产品长什么样

**怎么操作**:
1. **先去 Pinterest / Dribbble 找 2-3 张你喜欢的同类截图**(关键!)
2. 复制 Prompt 3,粘贴 PRD
3. 把 Prompt + 参考截图喂给 v0 / Lovable / Bolt 等原型工具
4. 它会**先问视觉设计语言**(阶段一),不要直接让它做页面
5. 设计语言确认后,做最重要的一页(阶段二)
6. 那页定稿后,铺开其他页面(阶段三)

**关键心法**:
- 顺序不能反:**先视觉语言 → 一页定稿 → 再铺开**
- 看原型只看三件事:**信息结构对不对、核心流程顺不顺、缺没缺页面**
- 不要让 AI 一次性"画完所有页面",会失控

**耗时**:2-6 小时(含改稿)

**完成的标志**:点开能跑的可交互原型,核心流程能走通。

**容易踩的坑**:
- 跳过参考图直接让 AI 自由发挥 → 出来的东西"AI 味"重
- 一上来铺开所有页面 → 改起来累、风格不统一
- 改原型时不更新 PRD → 下一步 Claude Code 拿到的是旧合同

---

## PRD 回写

**什么时候触发**:原型阶段改了流程、加/砍了页面、调整了功能优先级——任何让旧 PRD 过时的改动。

**怎么操作**:
1. 复制 Prompt 4
2. 把旧 PRD、最新原型描述、本轮主要改动填进去
3. 拿到一份新 PRD,**用它替换旧的**,不要新旧并存

**为什么必须做**:Claude Code 在 Step 4 读的是 PRD。PRD 过期 = 它做出来的东西跟原型对不上,白干。

---

## Step 4:Kickoff(项目开工)

**目标**:把 PRD + 原型交给 Claude Code,让它自己搭好项目骨架并写完 CLAUDE.md,**之后**才开始写功能。

**怎么操作**:

1. 在电脑上新建一个空文件夹(项目目录)
2. 把本工具包的 `CLAUDE.md` 复制进去,**占位符暂时不填**
3. 打开 Claude Code(在终端里 `cd` 进入项目目录,运行 `claude`)
4. 复制 Prompt 5,把 PRD、原型描述填进去,发给 Claude Code
5. Claude Code 会按四步走:
   - **步骤 1**:提议技术栈和目录结构 → 等你确认
   - **步骤 2**:搭建项目骨架(不写功能代码)→ 等你确认
   - **步骤 3**:自动填好 `CLAUDE.md` 上半"项目事实"
   - **步骤 4**:第一次 `git commit`,告诉你可以开始第一个功能
6. 之后每开发一个功能,告诉 Claude Code 要做什么,让它按 CLAUDE.md 里的规则做事

**耗时**:Kickoff 本身 30-90 分钟

**完成的标志**:项目能跑起来(空界面也算)+ `CLAUDE.md` 上半填完 + 第一次 git commit 完成。

**容易踩的坑**:
- 急着让它写功能、跳过 Kickoff → 一开始就乱
- Claude Code 提议技术栈时不读、直接点头 → 后面发现选错了再换,代价高
- 忘了把 CLAUDE.md 放进去 → Claude Code 没规则可参考,各种自由发挥

## 如何使用 CLAUDE.md

### 它是什么

把它当成"给 Claude Code 的入职手册"。每次 Claude Code 开始工作前,会读这份文件,按里面的规矩做事。

文件结构:
- **上半:项目事实**(技术栈、命令等)—— Kickoff 阶段让 Claude Code 自己填
- **下半:工作规范**(11 条普适规则)—— 原样保留,所有项目通用

### 第一次怎么用

跟着 Step 4 走就行。你不用手填上半,Claude Code 会做。

### 日常维护

**`#` 键沉淀新规则**:在 Claude Code 输入框,以 `#` 开头输入一句话,Claude Code 会自动把这条规则并入 CLAUDE.md。

- 第一次它跑测试前没提示你,你说"以后跑测试前先问我"
- 你输入 `# 跑测试前必须先问用户确认`,它写进 CLAUDE.md
- 下次会话,它已经"懂"这条

**控制行数**:CLAUDE.md 每次会话都会被完整读取。文件越长,对话越贵越慢。
- 目标:< 150 行
- 超了:回去看哪些规则不再用,删掉
- 每个季度或每个大版本回顾一次

**只沉淀长期规则**:
- ✅ "所有 API 调用必须加 timeout"(长期适用)
- ❌ "今天的 bug 是因为 X"(临时上下文,别沉淀)

## 遇到不懂的词怎么办

> 这是这份指南最关键的一段。

打开 `CLAUDE.md`,你会看到 `lint`、`typecheck`、`Conventional Commits`、`force push`、`DRY`、`.gitignore` 这些词。
非技术背景的你大概率不全懂——**这很正常**。

**不要无视,不要畏难,去查。**

每查清楚一个,你就比上一刻更懂软件工程一点。这个过程一开始很慢,但越往后越快——
**这就是飞轮在转动**。

你查的不是字典,你是在为下一次项目积累认知。
**今天看不懂的词,三个月后会变成你随口就能用的工具。**

## 常见疑问 FAQ

**Q:我不会写代码,真的能跑通整个流程吗?**
A:能。整个流程的核心是"你想清楚、AI 实现"。Step 1-3 你完全不用碰代码;Step 4 之后 Claude Code 写代码,你审。遇到不懂的术语,查就完了。

**Q:工具这么多,我应该用哪个?**
A:第一次随便选一个能用的。没有"最好",只有"开始"。等用熟了再换。

**Q:Claude Code 写错了怎么办?**
A:这就是 CLAUDE.md 里"即时 git"的意义。`git revert` 回到上一个 commit,告诉它哪里错了,重新来。所以频繁 commit 不是麻烦,是你的保险绳。

**Q:我的项目和模板里的"项目事实"对不上怎么填?**
A:直接问 Claude Code:"这一项我该填什么?"它会基于你的项目实际情况建议。

**Q:CLAUDE.md 的"工作规范"有些我看不懂,要不要先读懂再开始?**
A:不用。先用起来,遇到具体场景再回头查。学习永远是"用-查-理解"的循环,不是"先全懂再用"。

**Q:第一次没跑通整个流程,是我哪里做错了吗?**
A:大概率没做错。第一次本来就慢、本来就挫败。重要的是把过程中的卡点和发现按 `#` 沉淀下来,下次就快一点。
