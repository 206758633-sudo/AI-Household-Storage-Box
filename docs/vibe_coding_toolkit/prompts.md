# Vibe Coding 全流程通用 Prompt 合集

5 条通用提示词,对应"调研 → PRD → 原型 → PRD 回写 → Kickoff"全流程。
所有 `[填写: ...]` 处替换为你的项目信息后,即可直接使用。

---

## Prompt 1:行业 & 竞品调研

**用途**:Step 1 调研
**适合工具**:Kimi 的 agent 集群、ChatGPT、Claude、Perplexity 等调研型 AI

```
你是一位行业研究员。我要做一款 [填写: 一句话产品方向] 的产品。
目标用户大致是 [填写: 用户群描述,不确定可写"待定,请你也帮我提议"]。

请帮我做行业与竞品调研,按以下四块结构化输出,不要只贴链接:

### 一、市场现状与主要玩家
- 市场规模(若公开数据有限,给出估算与来源)
- 头部玩家 / 代表性产品 3-5 个,每家一句话定位

### 二、各家核心卖点
- 每家产品的差异化优势,一句话总结
- 各自主打的功能或商业模式

### 三、用户最常抱怨的痛点
- 公开评论 / 应用商店 / 社区里的高频负面反馈
- 至少 5 条,带出处或场景描述

### 四、尚未被满足的空白机会
- 现有产品集体没解决好的问题
- 3-5 个潜在切入点,并简评切入难度

要求:
- 结论先行,每块开头先一句话总结
- 引用来源必须可核查,不编造
- 不确定的地方明确标"未确认"
- 全程中文输出
```

## Prompt 2:苏格拉底追问 → 输出 PRD

**用途**:Step 2 厘清需求,产出 PRD
**适合工具**:ChatGPT、Claude 等会话型 AI

```
你是一位资深产品经理。我有一个还很模糊的产品想法:
[填写: 一句话描述]

请用苏格拉底式追问帮我厘清这个想法。规则:

1. 一次只问我一个问题。绝不一次甩出多个问题。
2. 根据我的回答,决定下一个问题——像剥洋葱,一层一层往下挖。
3. 重点覆盖以下维度:
   - 给谁用(目标用户特征、典型场景)
   - 解决什么具体痛点(对方现在怎么解决、为什么不满意)
   - 如果只做一个核心功能,是哪个
   - 什么算成功(可衡量的指标)
   - 明确不做什么(避免一上来什么都想做)
4. 当你判断信息已经足够,主动停下来,输出一份结构化 PRD,包含:
   - 项目一句话定位
   - 目标用户与典型场景
   - 核心痛点
   - MVP 核心功能(只列 1 个最重要的)
   - 成功标准
   - 明确不做的事
5. 整个过程不要写代码、不要画原型,只追问与梳理。

现在,从你的第一个问题开始。
```

## Prompt 3:高保真原型生成

**用途**:Step 3 生成可点击原型
**适合工具**:v0、Lovable、Bolt、Figma Make、Claude 等可生成 UI 的工具
**前置**:已有 PRD;已从 Pinterest / Dribbble 找 2-3 张同类产品截图作视觉参考

```
你是一位资深产品设计师。基于以下 PRD,生成产品的高保真原型:

[填写: 粘贴 PRD 全文]

按三个阶段进行,每阶段完成后停下,等我确认:

### 阶段一:确定视觉设计语言
我会附上几张参考截图(Pinterest / Dribbble)。基于这些图:
1. 提炼共通的视觉风格(色系、字体气质、圆角、留白节奏、整体调性)
2. 用 3-5 个关键词总结这套设计语言
3. 给出色板(主色 / 辅助色 / 背景 / 文字)与字体方案

不要直接生成页面,先确认设计语言。

### 阶段二:做最重要的那一页
PRD 里 MVP 核心功能对应的是哪一页?只做这一页,做到位:
1. 完整的视觉细节(严格按阶段一的设计语言)
2. 真实的内容,不要用 Lorem ipsum 或占位文字
3. 可点击的核心交互
4. 做完告诉我设计细节,等我确认

### 阶段三:以这页为基准铺开其余页面
其余页面的组件 / 间距 / 配色全部对齐阶段二,不要每页发挥不同风格。
按 PRD 的功能与流程,生成完整页面集合。

其他要求:
- 不要一上来铺开所有页面——会失控
- 视觉决策一律对齐阶段一的设计语言
- 任何阶段有疑问,先问我再做
```

## Prompt 4:PRD 回写更新

**用途**:Step 3 末。原型经过调整后,把改动同步回 PRD,重塑成最新版
**适合工具**:ChatGPT、Claude 等会话型 AI

```
我们在原型阶段对产品做了一些调整。请基于:

- 旧 PRD:
[填写: 粘贴旧 PRD]

- 最新原型:
[填写: 粘贴原型链接 / 关键页面截图描述 / 主要流程]

- 本轮主要改动:
[填写: 简述这一轮原型阶段做了哪些调整,粗略即可]

帮我重塑一份最新版 PRD,要求:

1. 完全对齐当前原型,不保留任何已被推翻的旧设计
2. 结构与旧 PRD 一致(项目定位 / 目标用户与场景 / 核心痛点 / MVP 功能 / 成功标准 / 不做的事)
3. 文末加一段"本次主要变更",简明列出:
   - 新增了什么
   - 删除 / 推翻了什么
   - 改动的理由
4. 不要"补充式"地把新旧混着写——要的是一份能直接交给 Claude Code 的、唯一权威的版本

输出格式:Markdown。
```

## Prompt 5:Kickoff(项目开工)

**用途**:Step 4。已有最新 PRD + 原型,在 Claude Code 里启动开发
**前置**:
- 已新建空项目目录
- 已把本套 `CLAUDE.md` 模板放入项目根目录(占位符暂保留不填)
- 已准备好 PRD 与原型

```
你是 [填写: 项目名] 的工程协作助手。这是开工的第一步,不要写任何功能代码。

资料:
- 最新版 PRD:
[填写: 粘贴 PRD 或附文件路径]

- 原型:
[填写: 链接 / 关键页面截图 / 主要流程描述]

- 协作规则:见本项目根目录 CLAUDE.md(请先完整读完,严格遵守)

请按下面四步进行,每一步完成后停下,等我确认:

### 步骤 1:提议技术栈与目录结构
基于 PRD 与原型,提议:
- 主语言、主框架、数据库、包管理工具
- 关键依赖(列出你打算用的库,说明各自解决什么问题)
- 目录结构(主要目录及作用)

给出选型理由(为什么选 X 而不是 Y)。等我确认后再动。

### 步骤 2:搭建项目骨架
我确认后,只做脚手架:
- 初始化项目(package.json / pyproject.toml / 等)
- 创建目录结构
- 装好基础依赖
- 跑通"空项目能启动"的最小验证

不写任何业务代码。

### 步骤 3:补全 CLAUDE.md 的"项目事实"段
基于刚搭好的脚手架,编辑根目录 CLAUDE.md 上半的占位符:
- 项目定位 / 技术栈 / 目录结构 / 常用命令 / 环境变量与密钥 / 不要碰的文件

下半"工作规范"保持不动。

### 步骤 4:首个 git commit
- 若项目未 git init,先 git init
- 创建 .gitignore(确保 .env / secrets.* / node_modules / 等被忽略)
- 创建 .env.example(仅占位符,不写真值)
- git add . && git commit -m "chore: 项目初始化与协作规则就位"
- 输出当前项目结构树,告诉我可以开始第一个功能

完成后停下,等我指定第一个功能开发。
```

## 使用顺序速查

1. 想法萌芽 → **Prompt 1**(去 Kimi/ChatGPT 调研)
2. 看完调研有方向 → **Prompt 2**(被苏格拉底追问,产出 PRD)
3. 拿 PRD + 参考图 → **Prompt 3**(原型工具,三阶段生成)
4. 原型改了几轮 → **Prompt 4**(回 ChatGPT/Claude 更新 PRD)
5. PRD 和原型都定稿 → **Prompt 5**(Claude Code 里开工)
6. 进入功能开发循环,过程中按 `#` 键沉淀规则到 CLAUDE.md
